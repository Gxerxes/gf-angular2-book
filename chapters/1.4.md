## 1.4快速开始 ##
通过前面Angular2的简介，相信会对它有个大致的了解，也想具体的了解它的用法。在这里，我们会使用一个通讯录的简单例子，为大家展开介绍Angular2的环境搭建与一些功能。

在接下来的内容中，首先会介绍Angular2环境搭建所需要准备的一些工具环境，包括node.js，npm，webpack等。当准备工作完成，我们会介绍搭建Angular2开发环境，包括Angular2所依赖的一些库，webpack打包工具，以及通过命令将整个环境运行起来等。当环境搭建完成并且运行成功，我们就可以在此基础上开始开发了，这里会先编写一个简单的Hello World例子来开始我们的第一个Angular2应用程序。其次，会通过一个完整的通讯录例子给大家全面的介绍Angular2，包括通讯录的一些功能。相信在接下来的旅程中，我们会慢慢的揭开Angular2的神秘面纱，对它有个更清晰的了解。

### 1.4.2准备工作 ###

#### 安装node.js
在使用Angular2开发前，有必要先把准备工作做好。环境搭建是基于node.js，所以首先需要安装它。可以去官网（https://nodejs.org/en/download/）
下载安装，本例子使用的是v5.11.0版本（最好是最新版本）。安装完成node.js后，在环境变量中的path添加目录地址，然后在dos或者终端上输入node -v可以查看node的版本，高版本的node.js已经集成了npm，可以输入npm -v查看npm的版本。

#### 安装webpack
在通讯录例子中，我们采用webpack打包，当然，还可以使用其他的工具，例如：gulp，grunt，fis3等。这里我们对webpack进行全局安装，在dos或者终端上运行npm install webpack -g即可。项目中，还会会创建一个webpack.config.js的文件，里面的配置在后续中介绍。

#### 环境搭建
mac环境搭建：

应用目录结构
```
├─  package.json
├─ README.md
├─tsconfig.json
├─typings.json
├─webpack.config.js
│
├─node_modules ...
├─src
│  │  app.ts
│  │  app-router.ts
│  │  index.html
│  │  polyfills.ts
│  │  typings.d.ts
│  │  vendor.ts
│  │
│  └─app
│     │  seed-app.html
│     │  seed-app.ts
│

```
通过上面的目录结构，我们需要一些配置文件来支持整个项目。

1.配置tsconfig.json <br/>
新建一个tsconfig.json文件，它是TypeScript编译配置文件，通过一些设置来指导Typesript进行编译，放于项目根目录下，并输入以下配置
```
{
  "compilerOptions": {
    "module": "commonjs",
    "target": "es5",
    "outDir": "dist",
    "rootDir": ".",
    "sourceMap": true,
    "emitDecoratorMetadata": true,
    "experimentalDecorators": true,
    "moduleResolution": "node"
  }
}
```
其中，"module"：指定模块代码生成。"target":ECMAScript指定版本。

2.配置typings.json <br/>
Typescript编译器并不能原生的支持一些Javascript的环境变量以及语法,所以使用TypeScript类型定义文件来解决这些兼容性问题。<br/>
创建 typings.json 文件，放于项目根目录下，并输入以下配置
```
{
  "name": "quickstart",
  "version": false,
  "dependencies": {},
  "ambientDependencies": {
    "es6-collections": "registry:dt/es6-collections#0.5.1+20160316155526",
    "node": "registry:dt/node#4.0.0+20160412142033"
  }
}
```
其中，"ambientDependencies"：所依赖的环境，"es6-collections"：为了支持旧的浏览器，"node"：是为了在nodejs环境中引用的对象提供类型定义

3.添加我们需要的库 <br/>
在项目根目录下创建package.json文件

package.json列出应用程序所需要的依赖包和一些有用的脚本。

4.打包工具 <br/>
在项目根目录下创建webpack.config.js文件
```
var webpack = require('webpack');
var path = require('path');
module.exports = {
  entry: {
    'polyfills': './src/polyfills.ts',
    'vendor':    './src/vendor.ts',
    'app':       './src/app.ts',
  },

  output: {
    path: './dist',
    filename: '[name].bundle.js'
  },

  resolve: {
    root: [ path.join(__dirname, 'src') ],
    extensions: ['', '.ts', '.js']
  },

  module: {
    loaders: [
      //.ts files for TypeScript
      { test: /\.ts$/, loader: 'awesome-typescript-loader' }
    ]
  }
};
```
entry 是页面入口文件配置，output 是对应输出项配置（即入口文件最终要生成什么名字的文件、存放到哪里）。module.loaders 是最关键的一块配置。它告知 webpack 每一种文件都需要使用什么加载器来处理。resolve 配置，root：查找module的话从这里开始查找，extensions：自动扩展文件后缀名，意味着我们require模块可以省略不写后缀名。

到这里，项目所依赖的环境基本搭建完成，下面我们创建项目文件

在src目录下创建文件app.ts
```
import {bootstrap} from '@angular/platform-browser-dynamic';
import {APP_ROUTER_PROVIDERS} from './app-router';
import {SeedApp} from './app/seed-app';

bootstrap(SeedApp, [
  APP_ROUTER_PROVIDERS
])
.catch(err => console.error(err));
```
app.ts文件为app的入口文件，其中的APP_ROUTER_PROVIDERS是在路由文件app-router.ts中，提供项目的所有路由配置。

seed-app.ts文件内容
```
import {Component} from '@angular/core';
import {ROUTER_DIRECTIVES} from '@angular/router';

@Component({
  selector: 'seed-app',
  directives: [ROUTER_DIRECTIVES],
  templateUrl: 'app/seed-app.html',
  styleUrls: ['app/seed-app.css']
})

  constructor() {}
}
```

seed-app.html文件内容
```
<h3>
  Hello，world！
</h3>
```
这个html文件会在seed-app.ts中通过templateUrl: 'app/seed-app.html'被引用。

创建文件polyfills.ts <br/>
在src目录下创建文件polyfills.ts
```
import 'core-js';
require('zone.js/dist/zone');
require('zone.js/dist/long-stack-trace-zone');
```
polyfills.ts文件主要是为项目引用zone.js，关于zone.js在后续章节中会详细的的介绍。

创建文件vendor.ts <br/>
在src目录下创建文件vendor.ts
```
import '@angular/platform-browser-dynamic';
import '@angular/platform-browser';
import '@angular/core';
import '@angular/http';
import '@angular/router';
```
vendor.ts文件将会引入项目所依赖的Angular2的基础包。

到此，基本环境搭建完成，就可以开始使用了，使用步骤：<br/>
1.在对应的项目根目录中，使用npm install命令，会根据package.json配置文件安装所依赖的包，安装的过程中，会自动在根目录上创建一个node_modules文件夹，项目依赖的包都在其中，该过程中，有的包安装可能失败，原因可能是连接不到外网，解决方法：使用淘宝的cnpm，或者是公司的镜像等<br/>
2.等依赖包安装完成，使用npm start命令启动服务<br/>
3.可以在浏览器中输入localhost:3000,看到效果<br/>

#### 整体功能
整个项目的交互，是由Angular2的路由（Router）完成的，在项目中，有一个单独的文件，位于src/app/router/app-router.ts，配置了项目的所有路由。配置如下：
```
import {RouterConfig, provideRouter} from "@angular/router";

import {ContactList} from "../components/contact-list/contact-list";
import {ContactDetail} from "../components/contact-detail/contact-detail";
import {Collection} from "../components/collection/collection";
import {Operate} from "../widget/operate";

export const ContactsAppRouters: RouterConfig = [
  { path: "", redirectTo: "contact-list", terminal: true },
  {
    path: "contact-list",
    component: ContactList
  },
  {
    path: "contact-detail/:id",
    component: ContactDetail
  },
  {
    path: "operate/id/:id",
    component: Operate
  },
  {
    path: "operate/isAdd/:isAdd",
    component: Operate
  },
  {
    path: "collection",
    component: Collection
  }
];

export const APP_ROUTER_PROVIDERS = [
  provideRouter(ContactsAppRouters)
];

```
RouterConfig中配置了所有路由，并且通过一个静态变量APP_ROUTER_PROVIDERS来抛出（export），这样我们就可以在app.ts文件中引入使用。

app入口文件（app.ts）
```
import {bootstrap} from '@angular/platform-browser-dynamic';

import {APP_ROUTER_PROVIDERS} from './app/router/app-router';
import {ContactApp} from './app/components/contact-app';


bootstrap(ContactApp, [
  APP_ROUTER_PROVIDERS
]);

```
在app.ts文件中，可以通过引用APP_ROUTER_PROVIDERS获得路由。路由配置并引用完成，页面间的跳转交互就可以实现了，接下来就是为每个页面填充对应的数据。

在每个项目中，数据的获取都是一个重点，Angular2中对数据的增删改查是通过其中的Http库实现的。在项目中，我们将数据交互的模块定位为service，在其他模块中引用service，就可以可以调用其中的方法，从而达到数据交互的目的。service的基本写法如下：
```
import {Injectable} from '@angular/core';
import {Http, RequestOptions, Headers} from '@angular/http';
import {Observable} from 'rxjs/Rx';
@Injectable()
export class ContactService {
  constructor(
    private _http:Http
  ) {}
  ...
}
```
其中@Injectable()标志着一个类可以被一个注入器实例化。关于Http服务的内容与依赖注入，在后续的章节中会着重介绍。

通常在页面间跳转会遇到传值到下一个页面使用的情况，在Angular2中页面间是通过router传值的，如下：
在app-router.ts中的详情页面对应的路由中，会在路由的后面加上‘/:id’，这样在页面跳转到详情页面时带上id这个参数：
```
{
  path: "contact-detail/:id",
  component: ContactDetail
}
```
在详情页面中就可以通过实例化的activatedRoute获取得到这个id的值：
```
this.activatedRoute.params.subscribe(params => {
      this.contact_id = params['id'];
      this.getById(this.contact_id);
    })
```
“activatedRoute”是通过import { ActivatedRoute} from '@angular/router'引用进来的，并在constructor(){}中实例化。
