## 1.4快速开始 ##
通过前面的简介，相信大家对Angular已经有了个大致的了解。在接下来的内容中，我们会使用一个简单的Hello World例子，为大家展开介绍Angular的环境搭建与功能。首先，Angular环境搭建所需要准备一些工具环境，包括node.js，npm，webpack等。其次，当准备工作完成我们会介绍搭建Angular开发环境，包括Angular所依赖的一些库，webpack打包工具等，当环境搭建完成我们就可以在此基础上开始开发了。接着，会编写一个简单的Hello World例子来开始我们的第一个Angular应用程序,以及通过命令将整个项目运行起来。最后，会通过一个完整的通讯录例子更加全面的介绍Angular，包括通讯录的一些功能。相信在接下来的旅程中，我们会慢慢的揭开Angular的神秘面纱，对它有个更清晰的了解。
### 1.4.1准备工作 ###
接下来，我们开始一步一步的搭建一个简单的Angular环境。
#### 安装node.js
在开始搭建环境前，我们需要做一些准备工作，包括Angular所依赖的基础环境node.js。所以在搭建环境之前我们需要先安装好node.js，可以去官网（https://nodejs.org/en/download/）
下载安装，本例子使用时node.js的版本是v5.11.0（最好是最新版本）。
安装完成node.js后，在环境变量中的path添加目录地址，然后在dos或者终端上输入node -v可以查看node.js的版本，高版本的node.js已经集成了npm，可以输入npm -v查看npm的版本。当可以看到node.js的版本后，说明机器上的node.js环境搭建成功，这样我们就可以继续往下搭建Angular环境了。

#### 环境搭建
当node.js安装完成，我们就可以正式的开始Angular项目环境搭建了。这里我们先来搭建一个最简单的Hello World例子并运行起来。

首先，创建一个新的文件夹，之后我们整个文件目录都存放于这个文件夹下。接着就可以开始我们的项目搭建了。Angular这个框架有一系列自己基础的依赖包，在项目运行起来前，就需要先把这些基础的依赖包下载到我们的根目录文件夹下，方便项目中引入，而这些包是怎么管理的呢？通常情况下，我们创建一个package.json的文件放在根目录下。package.json文件描述了一个NPM包的所有相关信息，包括作者、简介、包依赖、构建等信息，格式必须是严格的JSON格式。Angular的依赖包就配置在这个package.json中，文件内容如下：
```
{
  "name": "Hello world",
  "version": "1.0.0",
  "description": "A simple starter Angular project",
  "scripts": {
    "server": "webpack-dev-server --inline --colors --progress --display-error-details --display-cached --port 3000  --content-base src",
    "start": "npm run server"
  },
  "devDependencies": {
    "awesome-typescript-loader": "~0.16.2",
    "typescript": "~1.8.9",
    "typings": "^0.7.9",
    "webpack": "^1.12.9",
    "webpack-dev-server": "^1.14.0"
  },
  "dependencies": {
    "@angular/common": "2.0.0-rc.4",
    "@angular/compiler": "2.0.0-rc.4",
    "@angular/core": "2.0.0-rc.4",
    "@angular/http": "2.0.0-rc.4",
    "@angular/platform-browser": "2.0.0-rc.4",
    "@angular/platform-browser-dynamic": "2.0.0-rc.4",
    "@angular/router": "3.0.0-beta.2",
    "@angular/upgrade":  "2.0.0-rc.4"
  }
}
```
看完上面package.json的内容配置，这里简单的介绍下每个字段的意义。“name”：项目的名称，"version"：项目的版本，"description"：项目的简单描述，"scripts"：通过设置这个可以使NPM调用一些命令脚本，封装一些功能，"devDependencies"：项目开发是所依赖的的一些工具包，"dependencies"：项目依赖的基础Angular包。

在上一个章节中，大家对TypeScript有了个简单的认知。Angular选择TypeScript作为官方构建语言，但是TypeScript并不能被所有浏览器兼容，从而我们就需要有工具将TypeScript编译成JavaScript，让绝大部分浏览器识别。这里，我们创建一个tsconfig.json放在根目录下，它是TypeScript编译配置文件，通过一些设置来指导TypeSript进行编译，配置如下：
```
{
  "compilerOptions": {
    "module": "commonjs",
    "target": "es5",
    "outDir": "dist",
    "rootDir": ".",
    "sourceMap": true,
    "emitDecoratorMetadata": true,
    "experimentalDecorators": true,
    "moduleResolution": "node"
  }
}
```
其中，"module"：指组织代码的方式,"target":编译目标平台（es3,es5,es2015等）,"sourceMap":编译文件对应关系,"outDir":输出目录。通过tsconfig.json的配置，可以指导TypeSript进行编译，但编译器并不能原生的支持一些Javascript的环境变量以及语法,所以使用TypeScript类型定义文件来解决这些兼容性问题，所有我们创建一个typings.json文件放在根目录下，配置如下：
```
{
  "name": "quickstart",
  "version": false,
  "dependencies": {},
  "ambientDependencies": {
    "es6-collections": "registry:dt/es6-collections#0.5.1+20160316155526",
    "node": "registry:dt/node#4.0.0+20160412142033"
  }
}
```
其中，"ambientDependencies"：所依赖的环境，"es6-collections"：为了支持旧的浏览器，"node"：是为了在nodejs环境中引用的对象提供类型定义。

到这里，Hello World项目的基本配置文件搭建完成，接下来创建一个src文件夹，这个文件夹存放我们的业务代码文件。最终当项目运行起来后，浏览器上回打印出Hello World，这个显示的内容我们可以创建seed-app.html的模板文件，内容如下：
```
<h3>
  Hello，world！
</h3>
```
接着在src目录下创建文件seed-app.ts：
```
import {Component} from '@angular/core';

@Component({
  selector: 'seed-app',
  templateUrl: 'seed-app.html'
})
export class SeedApp {
}
```
在这个文件内容中，我们首先通过import从Angular的基础包core中引入组件（Component），通过组件我们可以定义该组件的dom元素名称，如：selector: 'seed-app'，也可以给组件引入所需要的模板，如：templateUrl: 'seed-app.html'。最后，通过类的形式定义该组件对外输出的名称，这样在其他的文件中就可以通过这个类名引用该组件。关于组件的具体内容可以在2.2章节中深入了解学习。

在Angular项目中，都有一个自己的入口文件，通过这个文件串联起整个项目，在根目录下，创建一个这样的入口文件app.ts，其内容如下：
```
import {bootstrap} from '@angular/platform-browser-dynamic';
import {SeedApp} from './seed-app';

bootstrap(SeedApp);
```
app.ts文件引入了两样东西来启动本应用：
1. Angular 的浏览器 bootstrap( 引导 ) 函数
2. 应用的根组件： SeedApp </br>
然后，调用 bootstrap 函数，并且把 SeedApp 传进去。

在项目中，是需要要引入Angular的基础包，这样才能驱动整个项目的运行，这里我们创建一个vendor.ts的文件放在根目录，内容如下：
```
import '@angular/platform-browser-dynamic';
import '@angular/platform-browser';
import '@angular/core';
import '@angular/http';
import '@angular/router';
```
最后，在根目录下创建index.html页面:
```
<!DOCTYPE html>
<html>
  <head>
    <meta charset=UTF-8>
    <meta name="viewport" content="width=device-width, initial-scale=1">
    <title>Angular Seed</title>
    <base href="/">
  </head>
  <body>
    <seed-app>
      Loading...
    </seed-app>
    <script src="vendor.ts"></script>
    <script src="app.js"></script>
  </body>
</html>
```
其中的seed-app元素就是我们在根组件SeedApp中定义的（selector: 'seed-app'）。到这里，我们整个Hello World项目基本搭建完成，但是我们怎样将这个项目运行起来呢？首先需要有打包工具，将项目打包编译。这里我们采用webpack来完成打包编译的工作，当然，还可以使用其他的工具，例如：gulp，grunt，fis3等。在之前的package.json文件的中，有这样一段配置：
```
"devDependencies": {
  ...
  "webpack": "^1.12.9",
  "webpack-dev-server": "^1.14.0"
}
```
其中，“webpack”就是我们的打包工具，"webpack-dev-server"则是webpack是一个小型的node.js Express服务器，这样项目就可以在这个服务器上运行。webpack这个打包工具也有自己的配置文件，在根目录下创建一个webpack.config.js文件，配置如下：
```
var webpack = require('webpack');
var path = require('path');
module.exports = {
  entry: {
    'vendor':     './src/vendor.ts'
    'app':       './src/app.ts',
  },

  output: {
    path: './dist',
    filename: '[name].js'
  },

  resolve: {
    root: [ path.join(__dirname, 'src') ],
    extensions: ['', '.ts', '.js']
  },

  module: {
    loaders: [
      //.ts files for TypeScript
      { test: /\.ts$/, loader: 'awesome-typescript-loader' }
    ]
  }
};
```
最后，当项目搭建完成，目录结构如下：
```
├─package.json
├─tsconfig.json
├─typings.json
├─webpack.config.js
└─src
   │  app.ts
   │  index.html
   │  seed-app.html
   │  seed-app.ts
   │  vendor.ts
```

到此，Hello World项目搭建完成，就可以开始运行，运行步骤：<br/>
1.在对应的项目根目录终端中，运行npm install命令，会根据package.json配置文件，安装所依赖的包。安装的过程中，会自动在根目录上创建一个node_modules文件夹，项目依赖的包都在其中。该过程中，有的包安装可能失败，原因可能是连接不到外网，解决方法：使用淘宝的cnpm，或者是公司的镜像等<br/>
2.等依赖包安装完成，使用npm start命令启动服务<br/>
3.可以在浏览器中输入localhost:3000（默认端口号为3000）,看到效果，如下：<br/>

在通讯录项目中，主要分为三大模块，联系人列表页：contact-list，联系人详情页：contact-detail，收藏页：collection。
在三大模块中，页面的跳转交互是怎么实现的呢？在Angular中提供了一个router的基础包，它就是路由，用来完成页面间的跳转。在项目中，创建有一个单独的文件，位于src/app/router/app-router.ts，配置了项目的所有路由，内容如下：
```
import {RouterConfig, provideRouter} from "@angular/router";

import {ContactList} from "../components/contact-list/contact-list";
import {ContactDetail} from "../components/contact-detail/contact-detail";
import {Collection} from "../components/collection/collection";
import {Operate} from "../widget/operate";

export const ContactsAppRouters: RouterConfig = [
  {
    path: "",
    redirectTo: "contact-list",
    terminal: true
  },
  {
    path: "contact-list",
    component: ContactList
  }
  ...
];

export const APP_ROUTER_PROVIDERS = [
  provideRouter(ContactsAppRouters)
];

```
配置好的路由可以在app.ts文件中引入使用。具体路由的介绍可以在2.7章节中深入的学习。接下来我们看下app.ts，它是整个项目的入口文件，内容如下：

```
import {bootstrap} from '@angular/platform-browser-dynamic';

import {APP_ROUTER_PROVIDERS} from './app/router/app-router';
import {ContactApp} from './app/components/contact-app';


bootstrap(ContactApp, [
  APP_ROUTER_PROVIDERS
]);

```
可以看到，在app.ts文件中，通过引用APP_ROUTER_PROVIDERS获得路由，这样页面间的跳转交互就可以实现了。接下来就是为每个页面填充对应的数据。

在每个项目中，数据的获取都是一个重点，Angular中对数据的增删改查是通过其中的服务（Service）实现的。在项目中，我们将数据交互的模块定位为service，在其他模块中引用service，就可以可以调用其中的方法，从而达到数据交互的目的。service的基本写法如下：
```
import {Injectable} from '@angular/core';
import {Http, RequestOptions, Headers} from '@angular/http';
import {Observable} from 'rxjs/Rx';
@Injectable()
export class ContactService {
  constructor(
    private _http:Http
  ) {}
  ...
}
```
其中@Injectable()标志着一个类可以被一个注入器实例化。关于服务的内容，在后续的2.5章节中会着重介绍。
